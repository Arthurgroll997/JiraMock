🔗 Pipeline Engine

YAML-basierte Cross-System Orchestrierung

# onboarding-pipeline.yaml
name: employee-onboarding
steps:
  - name: create-ad-user
    connector: active-directory
    action: createUser
    params:
      username: ${"${user}"}

  - name: provision-pam
    connector: fudo-pam
    action: createUser

  - name: create-ticket
    connector: matrix42-esm
    action: createTicket
▶️ Execute
POST /pipelines/run mit Variables und optionalem Dry-Run
✅ Validate
POST /pipelines/validate prüft YAML-Syntax und Connector-Verfügbarkeit
📜 History
GET /pipelines/runs zeigt alle vergangenen Pipeline-Ausführungen
🧪 Dry-Run
Testet den gesamten Workflow ohne Side-Effects

🚀 Getting Started

Drei Befehle. Eine Minute. Fertig.

# Clone the repo
$ git clone https://github.com/BenediktSchackenberg/PAMlab.git

# Start all 7 services + Studio
$ docker compose up -d

# Open PAMlab Studio
$ open http://localhost:3000
